# New Real-Time Model Team Onboarding Checklist

## Before onboarding starts
- [ ] use case and SLA defined
- [ ] named technical owner
- [ ] named business owner
- [ ] fallback behavior defined
- [ ] producer contracts identified

## Before stage access
- [ ] repo follows approved project style
- [ ] config, thresholds, and runbook structure in place
- [ ] replay validation path exists
- [ ] alarms and dashboards mapped
- [ ] rollback target identified

## Before prod launch
- [ ] stage replay evidence attached
- [ ] shadow/canary strategy approved
- [ ] threshold config reviewed by business owner
- [ ] first 24-hour watch plan published
- [ ] incident contacts confirmed
