# Ownership Map - Fraud Real-Time Platform

## Platform / MLOps owns
- deployment workflows
- endpoint config standards
- autoscaling policies
- release evidence templates
- runbooks and alarm wiring
- rollback tooling

## Fraud DS owns
- model logic
- evaluation interpretation
- challenger/candidate analysis
- feature recommendations with feature platform partners

## Fraud Ops owns
- threshold acceptability
- launch timing from business perspective
- review queue risk decisions
- business stop-loss consultation

## Data / Feature Platform owns
- producer contracts
- feature freshness pipelines
- replayability of source events
- online/offline feature consistency framework

## Security / Compliance owns
- approval expectations for regulated changes
- IAM and audit controls
- evidence retention expectations
