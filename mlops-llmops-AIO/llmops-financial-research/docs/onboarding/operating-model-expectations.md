# Operating Model Expectations for New LLM Teams

## You own
- domain-specific evaluation logic
- prompt quality proposals
- user feedback interpretation
- product-specific refusal and citation expectations

## The platform owns
- deployment and rollback workflows
- baseline monitoring and alerting
- approved repo/project structure
- IAM and control alignment

## Shared responsibilities
- canary decision making
- incident response on user-visible degradations
- cost/latency tradeoff reviews
- exception-path proposals
