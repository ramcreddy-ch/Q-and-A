# New LLM Product Team Onboarding Checklist

## Before onboarding starts
- [ ] named technical owner
- [ ] named business/product owner
- [ ] approved corpus defined
- [ ] expected user groups and entitlements defined
- [ ] latency and cost expectations defined

## Before stage access
- [ ] repo created from approved project style
- [ ] prompt versioning pattern adopted
- [ ] evaluation dataset identified
- [ ] ACL metadata strategy documented
- [ ] rollback prompt/model versions defined

## Before prod launch
- [ ] citation and groundedness eval attached
- [ ] entitlement tests passed
- [ ] runbooks reviewed
- [ ] alerting enabled
- [ ] canary and rollback owner assigned
